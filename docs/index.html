把浏览器变成 Mac 显示器

在 Mac 端选择屏幕、窗口或 BrowserDisplay 虚拟屏，通过 WebRTC 推送到同一局域网里的浏览器。手机、平板、另一台电脑，都能成为一块临时副屏。

开始使用 了解优势

Quick start

三步连接

  1. 选来源 屏幕、窗口，或独立虚拟屏。
  2. 浏览器打开 扫码或输入 WebViewer 地址。
  3. 开始传输 按网络状况选择画质。

Why BrowserDisplay

核心优势

用 BrowserDisplay 处理捕获、连接、画质和虚拟屏，浏览器只负责接收画面。

01

设备随手接入

手机、平板、电脑浏览器都能接收。

02

专用工作区

虚拟屏只展示你拖进去的窗口。

03

接收端免安装

浏览器打开 WebViewer 即可连接。

04

低延迟链路

ScreenCaptureKit 捕获，WebRTC 传输。

05

画质可控

从稳定到高清，按网络一键切换。

06

状态可见

端口、连接数、配对码和画质在一处显示。

Architecture

它如何工作

链路只在本地网络内建立，Mac 负责采集和发送，浏览器负责接收。

连接失败

确认 Mac 与接收设备在同一局域网，并使用 BrowserDisplay 显示的 WebViewer 地址。

找不到捕获源

重新授予屏幕录制权限后，退出并重新打开 BrowserDisplay。

画面卡顿

切到 1080p30 或 720p30，优先使用 5GHz / 6GHz Wi-Fi。

Experimental mode

虚拟扩展屏实验模式

不必捕获主屏。创建一块专用显示器，再发送到浏览器。

什么时候使用

演示备注、直播控屏、调试面板、会议共享。

外部依赖

由 BetterDisplay 提供虚拟显示能力。

安全边界

只清理 BrowserDisplay 自己创建的屏幕。

BetterDisplay CLI 示例

创建一块 16:9 HiDPI 虚拟屏

/Applications/BetterDisplay.app/Contents/MacOS/BetterDisplay create \
  -type=VirtualScreen \
  -virtualScreenName=BrowserDisplay-AB12 \
  -aspectWidth=16 \
  -aspectHeight=9 \
  -virtualScreenHiDPI=on \
  -resolutionList=1920x1080,2560x1440
